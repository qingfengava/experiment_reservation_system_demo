# 实验预约系统（Reservation System Demo）

## 一、项目简介

本项目是一个基于 **Qt 6 + C++** 实现的桌面实验预约管理系统，面向 **管理员、教师、学生** 三种角色，完成实验发布、预约、成绩管理等完整业务流程。

系统采用 **分层设计思想**，以文件作为持久化存储方式，结构清晰，逻辑完整，适合作为软件工程 / C++ / Qt 课程设计项目。

---

## 二、系统角色与功能概览

###  管理员（Admin）

- 实验管理
  - 添加 / 修改 / 删除实验
  - 发布实验（控制学生是否可预约）
- 用户管理
  - 学生管理（增删改查）
  - 教师管理（增删改查）
- 数据导出
  - 实验 / 学生 / 教师数据导出为 CSV / TXT
- 系统退出自动保存所有数据

---

### 教师（Teacher）

- 查看本人负责的实验列表
- 查看某实验的学生预约情况
- 成绩管理
  - 从 TXT 文件批量导入成绩
  - 手动修改学生成绩
- 成绩导出
  - 导出为 CSV / TXT
- 系统退出自动保存数据

---

### 学生（Student）

- 查看可预约实验（仅显示已发布实验）
- 预约实验 / 取消预约
- 查看本人预约记录
- 查询实验成绩
- 修改个人信息
- 导出预约记录（CSV / TXT）

---

## 三、程序亮点 

###  1. 清晰的分层架构设计

项目采用典型的 **MVC / 分层思想**：


- UI 不直接操作数据
- 业务逻辑集中在 Service 层
- 便于维护、扩展和调试

---

###  2. 基于文件的轻量级持久化方案

- 使用 TXT / CSV 文件保存数据
- 启动时自动加载数据
- 退出时统一保存数据
- 支持数据导入与导出，方便检查和调试

---

###  3. 健壮的数据解析与容错机制

- CSV 文件读取时进行：
  - 行校验
  - 时间格式校验
  - 数字字段合法性校验
  - 发布状态合法性校验
- 发现错误行自动跳过并输出调试信息
- 避免程序因脏数据崩溃

---

###  4. 发布机制设计合理

- 实验必须由管理员 **发布** 后
- 学生端才能看到并预约
- 符合真实教学管理系统逻辑

---

###  5. 良好的 Qt 编程规范

- 使用 `signals / slots` 绑定按钮功能
- 使用 `QDateTime`、`QTableWidget`、`QFileDialog` 等 Qt 组件
- 合理处理 `closeEvent`，确保数据安全

---

## 四、程序运行逻辑说明

1. **程序启动**
   - 设置工作目录
   - 从文件中加载学生、教师、实验、预约数据

2. **用户登录**
   - 根据账号类型进入不同界面

3. **业务操作**
   - 所有界面操作 → 调用 Service 层
   - Service 层操作内存数据结构

4. **数据保存**
   - 程序关闭时统一保存所有数据
   - 支持手动导出 CSV / TXT
